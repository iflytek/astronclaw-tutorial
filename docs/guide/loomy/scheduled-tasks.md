# 定时任务与积分说明

## 定时任务
定时任务可以让 Loomy 在指定时间自动执行固定流程，帮助你把重复性的工作交给 AI 持续处理。

**运行前提**
定时任务要正常执行，必须保证 Loomy 处于后台运行状态。如果应用被完全退出、系统强制结束进程，或者相关后台能力被关闭，定时任务将无法按计划触发。

![定时任务设置](/loomy/scheduled-tasks/mission.png)

### 适合什么任务
定时任务尤其适合那些“每天都要做”“每周都要做”或者“固定时间提醒”的工作，例如：
*   定时整理邮件摘要
*   周期性生成日报、周报
*   固定时间检查任务状态
*   在特定时间触发内容整理或信息收集

### 基本能力
通过定时任务，你可以提前定义好执行内容和触发时间，让 Loomy 在满足条件时自动运行，而不需要你每次手动重复下达指令。

一个完整的定时任务通常包含：
*   任务名称
*   触发时间或执行周期
*   执行指令或 Prompt
*   依赖的工具和权限
*   执行结果的查看方式

要让这些任务按时触发，除了任务配置本身正确之外，也需要确保 Loomy 在设备上持续后台运行。

### 使用建议
为了让定时任务更稳定，建议在创建时尽量把任务描述写清楚，特别是：
*   希望 Loomy 做什么
*   结果输出到哪里
*   是否需要依赖某个工具
*   出错时你期望如何处理

> **创建建议：**
> 如果一个任务涉及多步操作或外部工具调用，建议先**手动跑通一次**，再设置为定时任务，以降低自动执行时的跑偏概率。

### 常见问题

**定时任务适合处理复杂流程吗？**
可以，但前提是流程足够清晰、权限和工具都已经配置完成。越明确的任务，自动执行效果通常越稳定。

**定时任务失败后怎么办？**
建议优先检查任务描述、触发条件、模型配置和工具授权状态。很多失败都来自依赖未配置完整，或者任务描述不够具体。

**为什么到了时间没有自动执行？**
最常见的原因是 Loomy 没有在后台持续运行，或者相关权限、工具依赖没有准备完整。建议优先确认应用状态，再检查任务配置和执行条件。

## 积分体系

积分用于支持 Loomy 默认模型服务和相关核心能力调用。在你使用对话、任务执行或其他相关能力时，系统会根据对应规则消耗积分。

![积分说明](/loomy/credit/credit.png)

### 积分有什么用
积分主要用于支撑 Loomy 在执行任务过程中的资源消耗，包括但不限于默认模型调用、任务处理和部分高价值能力的使用。

**使用自定义模型时是否扣积分**
如果你在 Loomy 中配置并使用自己的 AI 模型服务提供商 API Key，相关模型调用通常不会消耗 Loomy 积分。
也就是说：
*   使用 Loomy 默认提供的模型服务时，会按照对应规则消耗积分
*   使用你自己配置的模型服务提供商时，模型调用本身不会扣除 Loomy 积分

具体是否涉及其他能力消耗，以产品内当前规则和实际功能说明为准。

> **API Key 安全说明**
> 你自行配置的 API Key 只会保存在本地设备中，Loomy 不会托管、上传或泄露这些密钥信息。
> **妥善保管你的密钥**：API Key 属于敏感信息，请不要随意分享给他人，也不要在不可信环境中保存或传输。若密钥泄露，可能会导致第三方冒用你的模型服务额度或访问能力。

### 如何获得积分
你可以通过以下方式获得积分：
*   申请加入 Waiting List 并通过审核
*   分享给你的好友。好友通过你的分享加入后，你和你的好友将分别获得 `5000` 积分
*   参与官方活动或内测
*   通过积分包充值方式获取，具体价格和积分包内容请以对应页面的实际展示信息为准

![获取积分](/loomy/credit/credit-add.png)

具体的发放方式和规则，以产品内当前展示的信息为准。

### 如何查看积分
你可以在 Loomy 客户端中查看当前积分余额以及历史变动记录。常见可查看的信息包括：
*   当前剩余积分
*   获得时间
*   获得方式
*   消耗记录

### 5000 积分可以做什么
`5000` 积分通常已经可以支持一批常见的办公任务。对于一般强度的办公场景，大致可以执行 `50` 次左右的任务。

例如，一个相对完整的办公任务链路可能包括：
1.  围绕“项目立项”主题，通过搜索引擎搜索和整理多类信息
2.  将搜索结果汇总成报告并保存到本地
3.  根据报告撰写邮件，向领导申请项目预算
4.  后续围绕项目细节，与领导进行约 `10` 次邮件往来沟通

类似这样的任务，`5000` 积分通常可以支持大约 `50` 次，具体会受到任务长度、上下文复杂度、调用模型次数以及是否联动外部工具等因素影响。

### 使用建议
如果你的任务属于多轮次、长时间、跨工具的复杂流程，通常会比普通单轮对话消耗更多积分。建议在执行前先确认任务目标和流程，避免无效消耗。
如果你已经有稳定可用的模型服务，也可以考虑自行配置 API Key，在部分场景下减少 Loomy 默认模型服务带来的积分消耗。

### 常见问题

**积分会在什么情况下消耗更快？**
通常是多步骤任务、长上下文任务、频繁调用模型或联动多个工具时，积分消耗会更明显。

**可以查看历史积分记录吗？**
可以。Loomy 会提供积分余额和相关变动记录，方便你了解积分的获得与消耗情况。

**使用自己的 API Key 会扣积分吗？**
不会。使用你自己配置的 AI 模型服务提供商 API Key 时，相关模型调用不会扣除 Loomy 积分，但请以当前产品规则和功能说明为准。
